MHL - Notekeeper Buffer Overflow

10 mins
Author
wetw0rk

I’m gonna be trying to exploit my first buffer overflow on the Android platform by attempting the NoteKeeper - Buffer Overflow challenge. In this challenge Mobile Hacking Labs provides you with the ability to use a Corellium device. However, I will be using an emulator, specifically a Small Phone API 31 Android 12.0 ("S") | arm64 emulated using Android Studio.

Installing the APK
#

The first thing we need to do is install the vulnerable APK we can do it using ADB as shown below:

wetw0rk@wetw0rks-MacBook-Pro APK Challenges % adb install com.mobilehackinglab.notekeeper.apk
Performing Streamed Install
Success

If everything went well you should be able to launch the app and see Note Keeper running.

Setting up LLDB
#

To debug the application (since I’m assuming this is a binary exploit), I would be using LLDB as android has officially dropped support for GDB.

The first thing you’re going to want to do is identify the server that you’ll be uploading to the target device. To do this we can use the locate command.

wetw0rk@wetw0rks-MacBook-Pro Desktop % locate lldb-server

/Applications/Android Studio.app/Contents/plugins/android-ndk/resources/lldb/android/x86_64/lldb-server

Playing with Application
#

So at this point I figured it would not hurt to play with the application directly. I quickly figured I’d spotted how we would be triggering the buffer overflow:

Howevever even after sending a crazy amount of A’s I still got no crash… Then I tried sending a bunch of B’s in the title vs the “note” area. and it looks like we got a crash!

Just looking at this it appears that we triggered a memcpy perhaps? Additionally we can see that this occured within libnotekeeper.so.
